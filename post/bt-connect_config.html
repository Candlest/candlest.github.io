    解决双系统蓝牙连接问题

    问题描述

    在双系统中，蓝牙设备在切换系统后无法连接。核心的问题是：

    每次蓝牙设备在一个双启动操作系统中配对后，它就会在另一个操作系统中停止工作。发生这种情况的原因是两个操作系统使用相同的蓝牙适配器和相同的 MAC。每个配对过程都会为适配器的 MAC 生成新的配对密钥。这样，在另一个操作系统中保存的先前配对密钥就会过时。

    解决方案

    思路说明

    对于使用 5.1 协议的设备，我们要同时解决下面两个问题：

    • 同步三个密钥，分别是 LinkKey、LEKey 和 LongTermKey。
    • 现代 BLE 设备在每次新配对时都会更改其 MAC 地址，所以我们还要同步 MAC 地址

    对于使用 3.0 协议的设备，我们只需要解决一个问题：

    • 同步 LinkKey

    恰好我的所有设备都支持 3.0 协议，故使用 3.0 版本的方法。

    实操过程

    从 Windows 读取注册表

    先下载 PsTools 置于 C:/Windows/ 目录下。

    打包 Bluetooth 相关的注册表：

    psexec -s -i regedit /e c:\keydump.reg HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\BTHPORT\Parameters\Keys

    不出意料，在适配器一节，会有如下的内容：

    [HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\BTHPORT\Parameters\Keys\1cce51cfb281]
"123414e9a685"=hex:10,ca,56,87,00,eb,2a,95,37,cb,3b,e1,b1,eb,79,f7

    其中的 "MAC"=hex:KEY 就是设备的 MAC 地址以及它的 LinkKey

    在 Linux 下更改蓝牙配置文件

    由于 MAC 地址不变，我们直接修改对应的文件：

    vim /var/lib/bluetooth/1C:CE:51:CF:B2:81/12:34:14:E9:A6:85/info

    每个设备的蓝牙适配器地址不一样，按照 /var/lib/bluetooth/<adapter>/<in-linux-paired-MAC>/info 修改上面的路径。

    找到 [LinkKey] 一节，使用 Win 下的 Key 覆盖原有的 Key：

    [LinkKey]
Key=10CA568700EB2A9537CB3BE1B1EB79F7
Type=4
PINLength=0

    在这里，hex:10,ca,56,87,00,eb,2a,95,37,cb,3b,e1,b1,eb,79,f7 被更改格式为 10CA568700EB2A9537CB3BE1B1EB79F7 。

    最后重启系统即可。

为什么不再用 Arch

出于对频繁更新和稳定性的考量，Arch 并不是一个适合长期使用的系统。从目前来看，除了 NixOS 这种可复现性强的系统设计，所有的操作系统都面临着增量更新导致的系统熵增问题。

更新快的系统的这种熵增会更加明显，因为在过去的六年内，我从来没有一次使用 Archlinux 在三个月内不重新安装系统。反而是 Ubuntu / Debian 安装后可以用到半年。

为什么不用 Ubuntu / Debian

使用过 pacman 后，不再能接受 apt 这样的复杂工具，apt autoremove 令我琢磨不透。

此外，Ubuntu 的系统一向不太干净，而 Debian 的软件更新速度太慢了！

为什么不用 NixOS

NixOS 学习曲线陡峭，文档混乱。且个人非常厌恶 /nix/store 目录下长串的、可读性差的文件夹，未来应该会有更好的解决方案。

为什么用 Fedora

我接触的第一个 Linux 系统是 CentOS，Rat Hat 的严谨作风让我印象深刻。

GNOME 和 Linux 的大金主是 Rat Hat。

综上所述，我的目光转向了 rpm 系的发行版。openSUSE 在国内并不太流行，那么 Fedora 作为 PC 的操作系统便是绝佳选择。

事实上也正是如此，Fedora足够稳定，足够方便，好用省心！

解决缩放/字体发虚问题

分数缩放解决不了问题，考虑使用字体缩放。

sudo dnf install gnome-tweaks

字体缩放设置为 1.5 ，采用 sarasa 系字体

安装字体

Fedora 的字体安装并不像 Arch 那样简单傻瓜化。

不考虑其他用户登陆的情况：

mkdir ~/.font

把下载好的字体文件夹拖入 ./font，运行：

fc-cache
